LingVa 隐私权政策

生效日期 / 最近更新：2026-07-17

适用产品：LingVa – Privacy-First Web Translator（Chrome 扩展程序）

开发者：MuShanYu

联系邮箱：mushanyu66@proton.me

1. 概述

LingVa 是一款轻量的浏览器翻译扩展，核心功能是翻译用户在网页上选中的文字，或在插件弹窗/悬浮窗中输入的文字。

LingVa 本身不运营任何后端服务器，不收集、不存储、也不向开发者上传任何用户数据。 本政策说明扩展在运行过程中会接触到哪些数据、这些数据流向何处，以及各项浏览器权限的用途，供 Chrome 网上应用店审核及用户参考。

2. 我们处理的数据类型

数据类型说明是否离开本机
待翻译文本 用户在网页上选中的文字，或在弹窗/悬浮球输入框中输入的文字 是——直接从用户浏览器发送至用户当前选用的翻译引擎（见第 4 节），LingVa 不经手、不留存
翻译结果 翻译引擎返回的译文 否，仅显示在界面并按需存入本机历史记录
翻译历史 最近的原文/译文记录，用于弹窗底部的历史列表 否，仅存储在本机 chrome.storage.local / localStorage，不同步、不上传
API 密钥 / 凭证 用户自行在设置中填写的第三方翻译服务凭证（百度 App ID/Token、Google API Key、微软 Subscription Key、有道 App Key/Secret、自定义接口地址与 Token） 否，仅存储在本机 chrome.storage.local；仅在用户主动发起翻译时被读取并用于向对应服务发起请求
界面偏好设置 显示语言、字体大小、主题模式、是否显示悬浮球、常用目标语言等 否，仅存储在本机

LingVa 不收集：用户身份信息、账号信息、位置信息、浏览历史、健康信息、财务信息，也不使用任何分析（Analytics）、埋点或第三方追踪 SDK。

3. 数据如何使用

上述数据仅用于实现翻译这一核心功能：

  • 待翻译文本被发送给用户选定的翻译引擎，用于生成译文；
  • 翻译历史、偏好设置、API 密钥仅用于在本机恢复用户的使用习惯与配置，不用于广告、画像、征信评估或任何与翻译功能无关的用途；
  • LingVa 不会将上述数据出售、出租或转让给任何第三方，也不会用于与提供翻译服务无关的目的。

4. 第三方翻译服务

LingVa 内置一个开箱即用的翻译引擎，并允许用户按需接入以下第三方翻译服务，或自定义翻译接口：

  • 内置翻译（基于 Google 非官方翻译端点）
  • 百度翻译开放平台
  • Google Cloud Translation API
  • 微软 Azure AI 翻译（Translator）
  • 有道智云翻译
  • 用户自行配置的自定义翻译接口

当用户选择上述任一引擎发起翻译时，待翻译文本会直接从用户浏览器发送至该服务商的服务器，该过程不经过 LingVa 或开发者的任何中间服务器。这些第三方服务如何处理数据，受其各自的隐私权政策约束，请用户在使用前自行查阅对应服务商的隐私政策。使用自定义接口时，数据将发送至用户自行填写的地址，其数据处理方式由用户自行负责。

5. 数据存储与保留

  • 翻译历史、设置项、API 密钥均通过 Chrome 提供的 chrome.storage.local 或页面 localStorage 存储在用户本机，不会同步至云端或开发者服务器。
  • 用户可以在插件设置中随时清空翻译历史、清除某个翻译引擎的已保存凭证，或点击「恢复默认设置」重置偏好设置。
  • 卸载 LingVa 扩展会连同上述所有本地数据一并删除。

6. 权限说明

LingVa 在 manifest.json 中声明了以下权限，均为实现翻译功能所必需：

权限用途
activeTab读取当前标签页中用户选中的文字，用于发起划词翻译；仅在用户主动选中文字时触发
storage将设置、翻译引擎凭证、本机翻译历史保存在本机，不涉及远程存储
scripting向网页注入划词翻译面板与悬浮球所需的界面脚本，仅渲染 UI，不修改页面原有内容
offscreenManifest V3 的后台 Service Worker 可能被系统随时终止，为避免翻译请求被中断，使用离屏文档维持网络请求直至返回结果
主机权限 <all_urls>划词翻译需要在任意网页生效，因此内容脚本需注入所有网址；同时部分翻译引擎与用户自定义接口需要跨域发起请求

7. 儿童隐私

LingVa 不专门面向 13 周岁以下儿童，也不会有意收集儿童的个人信息。
